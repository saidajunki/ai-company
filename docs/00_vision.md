# 目指していること（Vision）

## 一言で

Slackを統治UIとして、VPS上でAIがほぼ全作業を進める「有名で面白い存在になり、収益化を目指すAI会社」を作る。

## ゴール

- **Slackで会話するだけ**で、AIが調査・実験・実装・運用・記録・公開まで進める
- AIが「方向性がわからない」「権限が足りない」「契約が必要」などにぶつかったら、Slackで最小限の相談を上げる
- 10分ごとに、残コストと進捗が追える（展開が速い前提）
- 長期的に、月あたり約50万円規模の収益化につながる活動を積み上げる

## 重要な前提

- 有名になること・収益化を目的とする（ただし、迷走防止のため目的/評価軸は憲法に固定する）
- サイバー攻撃などは許容（“ボコボコにされても良いVPS”の前提）
- 後から運用状態を切り替える前提は置かない（発信したくなったら会社を作り直す想定）
  - 余計な概念は追加しない

## 公開について（実験前提）

- 実験の性質上、成果物は原則公開する（確認が簡単なため）
- ただし **APIキー/トークン/支払い情報/個人情報などの機微情報は公開しない**
  - 公開前チェックと鍵管理でリスクを制御する

## 事業内容の指向性

中央管理AIがシェルコマンドを実行できる前提で、**シェルで完結しやすい活動**に寄せる。

例:
- OSS/プロトタイプの作成、検証、公開
- 情報収集→要約→ノート化→公開（出典と日付つき）
- 生成した成果物の継続的改善（テスト/ベンチ/計測を含む）
